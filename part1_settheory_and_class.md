## 1.1〜1.2 論理記号と集合の記法

### モチベ

2部では「線形分離可能性」「関数完全性」のような定義を $\forall,\exists,\in$ などの記号で書く。先にこの記法を共通言語にしておかないと、2部の本文がただの暗号に見えてしまう。

### 定義・結果

**論理記号**

| 記号 | 読み方 | 意味 |
|:---:|:--|:--|
| $\land$ | かつ | 論理積（AND） |
| $\lor$ | または | 論理和（OR） |
| $\lnot$ | でない | 否定（NOT） |
| $\Rightarrow$ | ならば | 含意 |
| $\Leftrightarrow$ | 同値 | 双方向の含意（iff） |
| $\forall$ | すべての〜について | 全称量化子 |
| $\exists$ | 〜が存在する | 存在量化子 |
| $\exists!$ | ただ一つ存在する | 一意存在量化子 |
| $:\Leftrightarrow$ | 〜と定義する | 左辺を右辺で定義する記号 |

**集合の基本記法**

| 記号 | 意味 |
|:---:|:--|
| $x \in A$ | $x$ は $A$ の元である |
| $x \notin A$ | $x$ は $A$ の元でない |
| $A \subseteq B$ | $A$ は $B$ の部分集合 |
| $A \cap B$ | $A$ と $B$ の共通部分 |
| $A \cup B$ | $A$ と $B$ の和集合 |
| $A \setminus B$ | $A$ から $B$ を除いた差集合 |
| $\emptyset$ | 空集合 |
| $\{x \in A : P(x)\}$ | $A$ の元で性質 $P$ を満たすものすべて（内包表記） |

### 例

「$\forall x \in A\ \exists y \in B\ (x = f(y))$」は「$A$ のどの $x$ を取っても、$f(y)=x$ となる $B$ の $y$ が（少なくとも一つ）存在する」と読む。

AND ゲートの真理値表は内包表記で

$$\{(x_1,x_2) \in \{0,1\}^2 : x_1 = 1 \land x_2 = 1\} = \{(1,1)\} $$

と書ける。「$y=1$になる入力の集合」をこう表現できることが、2部の線形分離可能性の定義を読む基礎になる。

### 理由

$\forall x\,\exists y\, P(x,y)$ と $\exists y\,\forall x\, P(x,y)$ は意味が違う。前者は「$x$ ごとに $y$ を選んでよい」、後者は「全部の $x$ に共通して使える $y$ が一つある」。**量化子の順序は交換できない。** これが2部の定義を誤読しないための一番のポイント。

### サンプルコード（記法とPythonの対応）

内包表記 $\{x \in A : P(x)\}$ は、Pythonのリスト内包表記とそのまま対応する。

```python
A = [0, 1]
# {(x1,x2) in {0,1}^2 : x1=1 and x2=1}
result = [(x1, x2) for x1 in A for x2 in A if x1 == 1 and x2 == 1]
print(result)   # [(1, 1)]
```

「条件を満たす元だけを集める」という操作が、数学の記法とコードで完全に一致している。

### Q&A・演習

- 「OR ゲートの真理値表」を内包表記で書け。
- $\forall x\,\exists y\,(y > x)$（実数上）は真か偽か。$\exists y\,\forall x\,(y>x)$ はどうか。両者の違いを説明せよ。

## 1.3〜1.4 直積・冪集合・写像

### モチベ

パーセプトロンの入力 $x=(x_1,x_2)$ や重み $w$ が「どんな集合の元なのか」を正確に言いたい。$\mathbb{R}^n$ や $\{0,1\}^2$ という記法の意味を確認する。

### 定義・結果

- **順序対** $(a,b)$: 順番のついた2つの要素の組。$(a,b) \ne (b,a)$（$a\ne b$ のとき）。
- **直積** $A \times B := \{(a,b) : a \in A,\ b \in B\}$。
- **$n$ 重直積（冪）** $A^n := A \times A \times \cdots \times A$（$n$ 回）。
- **写像** $f: A \to B$ … $A$ の各元に $B$ の元をちょうど一つ対応させる規則。$A$ を**定義域**、$B$ を**値域**と呼ぶ。$f(x)$ をその**像**と呼ぶ。

### 例

$$\{0,1\}^2 = \{(0,0),\,(1,0),\,(0,1),\,(1,1)\} $$

これがAND/OR/NAND/XORの入力全体。$n=2$ の場合のパーセプトロンの入力 $x=(x_1,x_2)$ はこの集合の元として捉えている。$\mathbb{R}^n$ は実数の$n$重直積で、重み$w$や入力$x$が住む空間。

### 理由

パーセプトロンは写像 $y : \{0,1\}^2 \to \{0,1\}$ そのもの。$y(x_1,x_2)$ という数学の記法と、Pythonで `gate(x1, x2)` と書くことは、本質的に同じものを表している。これが2部でclassを導入する動機（$y=P(x)$ と `p(x)` を一致させたい）に直結する。

### サンプルコード（記法とPythonの対応）

$\{0,1\}^2$ のような直積は、Pythonでは `itertools.product` で生成できる。

```python
import itertools
list(itertools.product([0, 1], repeat=2))
# [(0, 0), (0, 1), (1, 0), (1, 1)]
```

- パーセプトロンが「写像」であるとはどういうことか、$f:A\to B$ の定義に沿って説明せよ。

## 1.5 量化子つきの命題を読む練習（2部の先取り）

### モチベ

2部で実際に出てくる2つの定義を、証明より先に「読み方」だけ確認しておく。記号の意味（1.1〜1.4）が分かっていても、複数の量化子が重なると急に読めなくなることが多いので、ここで一度練習する。

### 定義・結果

**(a) 線形分離可能性**

$$\exists\, w \in \mathbb{R}^n,\ \exists\, b \in \mathbb{R}\ \text{ s.t. }\ \forall i\ \bigl[(t^{(i)}=1 \Rightarrow w\cdot x^{(i)} + b > 0)\ \wedge\ (t^{(i)}=0 \Rightarrow w\cdot x^{(i)} + b \le 0)\bigr] $$

**(b) 関数完全性**

$$\forall n \in \mathbb{N},\ \forall f : \{0,1\}^n \to \{0,1\},\ \exists\, G\text{ の素子の非巡回結合 } C \text{ s.t. } C \equiv f $$

### 例

(a)の読み方: 「**全ての**データ点を正しく分類できる $w,b$ の組が**少なくとも一つ存在する**」。
(b)の読み方: 「**どんな**入力数 $n$、**どんな**ブール関数 $f$ を持ってきても、それを実現する回路 $C$ が**作れる**」。

### 理由

(a)は $\exists$ が先頭にあるので「$w,b$ を先に選んでから、全データに対して条件を確かめる」という順序。(b)は $\forall n\,\forall f$ が先頭にあるので、$n,f$ ごとに $C$ を選んでよい（$C$ が $n,f$ に依存してよい）。量化子の順序が意味を決める、という1.1の教訓がそのまま使われている。

### サンプルコード

ここは記法の読解が主役なので、コードでの検証は2部2.4（XORが線形分離可能かどうかを実際に探索する）に譲る。気になる人は先に`xor_gridsearch_demo.py`を覗いてもよい。

## 1.6 Python の class 文法

> 注: このセクションは友人が担当し、ここでは汎用的な例（パーセプトロンとは無関係）で class の文法を説明する。パーセプトロンに即した具体例・演習は2部で扱う。

### モチベ

「データを覚えておいて、呼ばれたら使う」だけなら、実は class は必須ではない（クロージャで同じことができる）。class が本当に効くのは、**同じデータ（状態）を読み書きする2つ以上の操作（メソッド）があり、そのデータが時間とともに変化する**とき。この基準を、パーセプトロンと無関係な例で先に身につける。

### 定義・結果

```python
class Dog:
    def __init__(self, name):   # コンストラクタ：インスタンス生成時に1回だけ呼ばれる
        self.name = name        # self.xxx = ... でインスタンス自身に属性を持たせる

    def bark(self):              # メソッド：第一引数は必ず self
        return f"{self.name}: Woof!"
```

- `class` … 型（クラス）を定義するキーワード。
- `__init__(self, ...)` … インスタンスが作られるときに自動で呼ばれる初期化メソッド。
- `self` … 「このインスタンス自身」を指す。メソッドの第一引数には必ず書く（呼ぶときには書かない）。
- `__call__` … これを定義すると、インスタンスを関数のように呼び出せる（`p(x)`のように）。
- **継承の基準**: サブクラスが親クラスに**新しいメソッド（振る舞い）を追加・変更**するなら継承を使う。コンストラクタの引数（データ）だけが違うなら、継承ではなく**インスタンス化**で十分。

### 例

```python
class BankAccount:
    def __init__(self, balance=0):
        self.balance = balance

    def deposit(self, amount):
        self.balance += amount

    def withdraw(self, amount):
        if amount > self.balance:
            raise ValueError("残高不足")
        self.balance -= amount
```

`deposit`と`withdraw`が同じ`self.balance`を共有しながら変化させる——classの典型的な使い道。

継承の正しい例と避けるべき例:

```python
class Animal:
    def __init__(self, name):
        self.name = name
    def speak(self):
        return f"{self.name} makes a sound"

class Dog(Animal):              # 新しい振る舞い(speakの上書き)を追加 -> 正しい継承
    def speak(self):
        return f"{self.name}: Woof!"

class USDAccount(BankAccount):   # currencyという「データ」が違うだけ -> 避けるべき設計
    def __init__(self, balance):
        super().__init__(balance)
        self.currency = "USD"
```

`USDAccount`は`BankAccount`と振る舞いが何も変わっていない。データが違うだけなら、サブクラスを作らず属性として持たせれば十分（`acc.currency = "USD"`）。

### 理由

`Doubler`のような「評価するだけ」のクラスはクロージャでも書けるが、`__call__`の文法を覚えるための最小例として使う。一方`BankAccount`は`deposit`/`withdraw`という複数の操作が状態を共有・変化させるので、クロージャでは`nonlocal`が必要になり崩れやすい——classが素直に勝つ例。

### サンプルコード（一緒に読む）

```python
class Doubler:
    def __init__(self, factor):
        self.factor = factor
    def __call__(self, x):
        return self.factor * x

double = Doubler(2)
double(5)   # 10

acc = BankAccount(1000)
acc.deposit(500)
acc.withdraw(200)
print(acc.balance)   # 1300
```
